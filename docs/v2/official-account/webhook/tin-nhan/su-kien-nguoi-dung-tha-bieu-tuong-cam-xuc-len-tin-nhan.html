Sự kiện người dùng thả biểu tượng cảm xúc lên tin nhắn

Lưu ý

Quyền cần có:  Sự kiện người dùng thả biểu tượng cảm xúc Sự kiện này chỉ được gửi về khi user nhận tin nhắn trên Zalo Mobile

Khi người dùng thả biểu tượng cảm xúc vào tin nhắn trong cửa sổ chat với OA, hệ thống Zalo sẽ gửi đến Webhook Url của Official Account một request http như sau

  • URL: Webhook Url của ứng dụng
  • Method: POST
  • Content Type: application/json
  • X-ZEvent-Signature: mac = sha256(appId + data + timeStamp + OAsecretKey), với data là chuỗi json trả về dưới đây.

Ví dụ:​

{
    "app_id": "4591324301967398637",
    "event_name": "user_reacted_message",
    "sender": {
        "id": "186729651760683225"
    },
    "recipient": {
        "id": "607812198688816074"
    },
    "message": {
        "msg_id": "789f573aecf8a9a4f0eb",
        "react_icon": ":-h"
    },
    "timestamp": "1637119864504",
    "user_id_by_app": "835572138790572528"
}

Mô tả thuộc tính​

Ví dụ:
{
    "id": "388613280878808645"
}
Trong đó:
  • “id”: Id của Official Account/ User nhận biểu tượng cảm xúc
Thuộc tínhKiểu dữ liệuMô tả
app_idStringId của ứng dụng bên thứ 3
senderJson ObjectVí dụ:
{
    "id": "246845883529197922"
}
Trong đó:
  • “id”: Id của User thả biểu tượng cảm xúc
recipientJson Object
event_nameStringGiá trị là “user_reacted_message”.
messageJson ObjectVí dụ:
{
    "msg_id": "43b59c025aebfb5e6fa",
    "react_icon": "/-strong"
}
timestampStringThời điểm request được gửi, tính bằng milisecond
user_id_by_appStringTrường này chỉ có khi app_id và oa_id trên đang liên kết với nhau.

Id này có thể sử dụng cho social_api
